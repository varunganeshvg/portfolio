Social Media Web Application

A social media platform inspired by Instagram, currently under development, focused on scalable backend architecture, authentication, and service separation using microservices.

Status: Under Development

Authentication Service

The authentication service is responsible for managing user identity and security across the platform. It handles user registration, login, and token-based authentication.

  • User registration with encrypted password storage
  • Login authentication using generated access tokens
  • Token validation for protected endpoints
  • Role-independent access for standard users

User Profile & Account Management

This module manages user profile information and account-related data. Each user has a dedicated profile that can be viewed and updated independently.

  • Profile creation and update
  • Public profile viewing
  • User-to-user relationship management (followers / following)

Post & Media Service

The post service manages user-generated content such as posts, images, and captions. Media uploads are handled separately to ensure performance and scalability.

  • Post creation with image uploads
  • Caption and metadata handling
  • Secure media storage and retrieval
  • Post visibility control

User Interactions

Interaction features allow users to engage with content and other users across the platform.

  • Like and comment functionality
  • Post sharing mechanisms
  • Feed generation based on user activity

System Architecture

The application follows a microservice-oriented architecture, where each core functionality is separated into independent services.

  • Authentication Service – handles login, tokens, and security
  • User Service – manages profiles and relationships
  • Post Service – handles posts, media, and interactions

This architecture allows independent development, easier scaling, and improved maintainability as the application grows.
